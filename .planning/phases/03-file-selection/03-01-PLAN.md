  <behavior>
**FILE-01: Random selection from source folder**
- Given a folder with multiple supported files, select_random_printable_file() returns one of them
- Calling select_random_printable_file() multiple times on same folder can return different files (randomness)
- Selection uses Python's random.choice() for uniform random distribution

**FILE-02: Extension filtering**
- Files with .txt extension are included in selection
- Files with .png extension are included
- Files with .jpg extension are included
- Files with .jpeg extension are included (JPEG allows both)
- Files with .bmp extension are included
- Extension matching is case-insensitive (.TXT, .PNG, .Txt all match)
- Files with unsupported extensions (.pdf, .doc, .mp3) are excluded
- Directories are excluded even if named like files ("test.txt" folder excluded)

**FILE-03: Empty folder handling**
- Empty folder (no files at all) returns None and logs warning
- Folder with only unsupported files returns None and logs warning
- Warning message includes folder path and list of supported extensions
- No exceptions raised for empty/no-match cases (graceful degradation)

**Integration:**
- Function accepts PrinterConfig and reads source_folder field
- Relative paths in source_folder are resolved to absolute paths
- Returns pathlib.Path objects (not strings) for type safety
  </behavior>

2. **Main function signature**:
```python
def select_random_printable_file(config: PrinterConfig) -> Path | None:
    """Select a random printable file from configured source folder.

    Requirements:
    - FILE-01: Random selection from source folder
    - FILE-02: Only supported file types (.txt, .png, .jpg, .jpeg, .bmp)
    - FILE-03: Empty folder logs warning, doesn't crash

    Args:
        config: PrinterConfig with source_folder field

    Returns:
        Path to randomly selected file, or None if no files available
    """
```

3. **Implementation approach (from RESEARCH.md patterns)**:
   - Extract source_folder from config, resolve to absolute path if relative
   - Use `folder.iterdir()` with list comprehension to filter files
   - Filter with `.is_file() and .suffix.lower() in SUPPORTED_EXTENSIONS`
   - Pre-check if list is empty (LBYL pattern for expected case)
   - If empty: `logger.warning(f"No printable files found in {folder}...")` and return None
   - If not empty: `return random.choice(files)`

4. **Imports needed**:

**Key patterns to follow (from Phase 02 SUMMARYs):**
- Use pathlib.Path throughout (not os.path string manipulation)
- Case-insensitive extension matching with `.suffix.lower()`
- Follow existing logging pattern: `logger = logging.getLogger(__name__)`
- Type hints with modern syntax: `Path | None` (Python 3.10+)
- Return None for "no file available" case (explicit null handling for caller)

**Anti-patterns to avoid (from RESEARCH.md):**
