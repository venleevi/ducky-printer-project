  <implementation>
    After tests pass, implement:

    1. **start_gpio_listener(config: PrinterConfig) function:**
       - Sets Button.pin_factory = LGPIOFactory() for Pi compatibility
       - Branches on config.trigger_mode ("press" vs "switch")
       - Returns Button or InputDevice object (caller manages lifecycle)

    2. **Button mode (trigger_mode="press"):**
       - Create Button(config.gpio_pin, pull_up=True, bounce_time=0.01)
       - Attach callback to button.when_pressed
       - Callback checks cooldown, calls handle_print_trigger(config)

    3. **Switch mode (trigger_mode="switch"):**
       - Create InputDevice(config.gpio_pin, pull_up=True, bounce_time=0.01)
       - Attach callbacks based on switch_direction:
         - "both": when_activated AND when_deactivated
         - "on_only": when_activated only
         - "off_only": when_deactivated only
       - Callbacks check cooldown, call handle_print_trigger(config)

    4. **Cooldown enforcement:**
       - Use time.time() to track last_trigger_time (module-level variable)
       - In callback: check if time.time() - last_trigger_time >= config.cooldown_seconds
       - If within cooldown: log debug message, return early (no print)
       - If cooldown expired: update last_trigger_time, call handle_print_trigger

    5. **Debounce:**
       - Pass bounce_time=0.01 (10ms) to Button/InputDevice constructor
       - gpiozero handles debounce in hardware (GPIO-04)

    6. **Error handling:**
       - Wrap callback in try-except
       - Log exceptions with logger.exception() (includes stack trace)
       - Never crash listener (matches Phase 04 error resilience)

    7. **Logging:**
       - Debug: "GPIO event ignored (within cooldown)"
       - Info: "GPIO trigger activated"
       - Error: "GPIO callback error: {exception}"
  </implementation>
