  <files>src/file_handler.py, tests/test_file_handler.py</files>
  <behavior>
    - Test 1: read_file("test.txt") with valid UTF-8 file returns content as string
    - Test 2: read_file("missing.txt") raises FileError with "File not found" message
    - Test 3: read_file() with binary file raises FileError with "not valid UTF-8" message
    - Test 4: read_file() with permission denied raises FileError with "Permission denied" message
    - Test 5: resolve_filepath("receipt.txt") with default folder returns /GEN26_BILLPRINTER/receipt.txt path
    - Test 6: resolve_filepath("/full/path.txt") returns path as-is without prepending base folder
  </behavior>
  <action>
    Create src/file_handler.py module with:

    1. Custom FileError exception class (inherits from Exception)

    2. resolve_filepath(filename, base_folder="/GEN26_BILLPRINTER") function:
       - Use pathlib.Path for path manipulation (not string concat - security best practice per RESEARCH.md)
       - If filename is absolute path, return as Path object
       - If relative filename, prepend base_folder
       - Return Path object

    3. read_file(filename, base_folder="/GEN26_BILLPRINTER") function:
       - Call resolve_filepath() to get full path
       - Open file with UTF-8 encoding in try/except block
       - Catch FileNotFoundError → raise FileError("File not found: {path}")
       - Catch UnicodeDecodeError → raise FileError("File is not valid UTF-8: {path}")
       - Catch PermissionError → raise FileError("Permission denied reading file: {path}")
       - Catch generic Exception → raise FileError("Error reading file {path}: {e}")
       - Return file content as string

    Implementation notes per CONTEXT.md decisions:
    - Assume files are in /GEN26_BILLPRINTER/ unless full path provided
    - Leave files unchanged after reading (no deletion or moving)
    - Support .txt files minimum (no file type filtering in this module - that's caller's responsibility)
    - Assume UTF-8 encoding as specified

    Per RESEARCH.md anti-patterns to avoid:
    - Do NOT use string path manipulation (use pathlib.Path)
    - Do NOT fail silently (raise clear FileError exceptions)

    Also create tests/test_file_handler.py with pytest test cases covering all behavior listed above. Use pytest.raises for exception testing. Use tmp_path fixture for test file creation.
  </action>

<task type="auto">
  <name>Task 3: Create USB permissions setup documentation</name>
  <files>docs/SETUP.md</files>
  <action>
    Create docs/SETUP.md with clear instructions for setting up USB printer permissions on Raspberry Pi.

    Include:

    1. **Prerequisites section:**
       - Raspberry Pi 3B+ or newer
       - Raspberry Pi OS (Debian-based)
       - Citizen CT-S310IIEBK thermal printer connected via USB
       - Python 3.7+ installed

    2. **Install dependencies section:**
       ```bash
       # System dependencies (libusb for USB communication)
       sudo apt-get update
       sudo apt-get install -y libusb-1.0-0 python3-pip

       # Python dependencies
       pip install -r requirements.txt
       ```

    3. **USB Permissions Setup section** (critical for non-root access):
       - Step 1: Find printer USB vendor/product ID with `lsusb` command (explain output format)
       - Step 2: Create udev rule file at /etc/udev/rules.d/99-thermal-printer.rules
       - Provide TWO rule options:
         a) Generic rule by USB class (class 7 = printer): `SUBSYSTEM=="usb", ATTR{bInterfaceClass}=="07", MODE="0666"`
         b) Specific rule by vendor/product ID (example for Citizen): `SUBSYSTEM=="usb", ATTR{idVendor}=="2730", ATTR{idProduct}=="XXXX", MODE="0666"`
       - Step 3: Reload udev rules: `sudo udevadm control --reload-rules && sudo udevadm trigger`
       - Step 4: Reconnect printer (unplug/replug USB cable)
       - Step 5: Verify permissions with `ls -l /dev/usb/lp*` or similar

    4. **Alternative: User group method** (if udev rules don't work):
       - Add user to lp group: `sudo usermod -aG lp $USER`
       - Log out and back in

    5. **Troubleshooting section:**
       - "Permission denied" errors → Check udev rules applied and printer reconnected
       - "No printer found" errors → Verify printer powered on and USB connected
       - "Invalid endpoint address" errors → May need to specify in_ep/out_ep (covered in future plan)

    6. **Testing connection section:**
       - How to test once print_job.py is ready (preview for future plans)
       - Example command: `python src/print_job.py test.txt`

    Use clear markdown formatting with code blocks, step numbers, and warning callouts for critical steps.

    Per RESEARCH.md Pitfall #2, USB permission setup is essential for non-root operation and a common failure point.
