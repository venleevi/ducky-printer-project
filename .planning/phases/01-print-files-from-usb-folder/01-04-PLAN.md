<task type="checkpoint:human-action" gate="blocking">
  <action>Install dependencies and setup USB permissions</action>
  <instructions>
    Follow the setup instructions from docs/SETUP.md:

    1. **Install system dependencies:**
       ```bash
       sudo apt-get update
       sudo apt-get install -y libusb-1.0-0 python3-pip
       ```

    2. **Install Python dependencies:**
       ```bash
       pip install -r requirements.txt
       ```
       Expected: python-escpos, pytest, pytest-mock installed successfully

    3. **Setup USB printer permissions:**

       a. Connect Citizen CT-S310IIEBK printer via USB cable to Raspberry Pi
       b. Power on the printer
       c. Run `lsusb` to verify printer is detected
          Expected output: Line showing Citizen Systems device
       d. Note the vendor ID (format: ID XXXX:YYYY)
       e. Create udev rule:
          ```bash
          sudo nano /etc/udev/rules.d/99-thermal-printer.rules
          ```
          Add line: `SUBSYSTEM=="usb", ATTR{bInterfaceClass}=="07", MODE="0666"`
          Save and exit (Ctrl+O, Enter, Ctrl+X)
       f. Reload udev rules:
          ```bash
          sudo udevadm control --reload-rules && sudo udevadm trigger
          ```
       g. Reconnect printer (unplug and replug USB cable)

    4. **Verify automated tests still pass:**
       ```bash
       pytest tests/ -v
       ```
       Expected: All tests green (file_handler, printer, print_job tests pass)

    **Common issues:**
    - "Permission denied" error: Check udev rule applied and printer reconnected
    - "No module named 'escpos'": Run `pip install -r requirements.txt` again
    - "libusb not found": Install libusb-1.0-0 via apt-get

    **If udev rules don't work:**
    Alternative method - add user to lp group:
    ```bash
    sudo usermod -aG lp $USER
    ```
    Then log out and back in.
  </instructions>
  <resume-signal>Type "setup-complete" when dependencies installed and USB permissions configured, or describe any issues encountered</resume-signal>
</task>
