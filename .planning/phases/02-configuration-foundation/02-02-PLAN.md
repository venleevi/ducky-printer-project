<feature>
  <name>YAML Configuration Loading with Validation</name>
  <files>src/config.py, tests/test_config.py</files>
  <behavior>
    Configuration loading behavior (from RESEARCH.md patterns):

    SUCCESS CASES:
    - Valid YAML with all fields → PrinterConfig object with values from file
    - Partial YAML (missing fields) → PrinterConfig with defaults for missing fields
    - Empty YAML file → PrinterConfig with all defaults
    - Custom config path provided → Loads from specified path
    - No custom path → Searches project root then ~/.config/ducky-printer/

    ERROR CASES:
    - Missing config file → FileNotFoundError listing all searched paths
    - Invalid YAML syntax (tabs, bad indentation) → ValueError with line/column number
    - GPIO pin < 0 or > 27 → ValueError explaining BCM range
    - GPIO pin is string not int → TypeError explaining expected type
    - File permission denied → FileNotFoundError with permission hint

    VALIDATION RULES:
    - gpio_enabled: boolean (default True)
    - web_enabled: boolean (default True)
    - target_file: string converted to Path object (default "wish1.png")
    - gpio_pin: integer 0-27 (default 17 for BCM pin 17 = physical pin 11)

    TYPE CONVERSIONS:
    - YAML string paths → pathlib.Path objects in __post_init__
    - Frozen dataclass → immutable configuration (thread-safe)
  </behavior>
