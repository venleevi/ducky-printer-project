  <action>
    Create printer_config.yaml in project root with all configuration settings and comprehensive inline comments.

    File structure (from RESEARCH.md example):

    ```yaml
    # printer_config.yaml - Thermal printer trigger configuration
    # Place this file in project root directory
    # See docs/CONFIGURATION.md for detailed documentation

    # Enable/disable GPIO button trigger
    # Set to false to disable physical button without code changes
    # Default: true
    gpio_enabled: true

    # Enable/disable web interface trigger
    # Set to false to run only GPIO trigger (headless operation)
    # Default: true
    web_enabled: true

    # Target file path for printing
    # Relative paths are relative to project root
    # Absolute paths also supported: /home/pi/images/wish1.png
    # Supported formats: .txt, .png, .jpg, .jpeg, .bmp
    # Default: wish1.png
    target_file: wish1.png

    # GPIO pin number for button (BCM numbering, NOT physical pin number)
    # BCM 17 = Physical pin 11 (see https://pinout.xyz)
    # Valid range: 0-27 for Raspberry Pi 3B+
    # Default: 17
    gpio_pin: 17
    ```

    Match exact values from PrinterConfig dataclass defaults. Include comment about BCM vs physical numbering (common confusion per RESEARCH.md Pitfall #7).

    File should be ready for user to copy and customize without modification (defaults work out of box).
  </action>
