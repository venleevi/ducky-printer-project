### 2. PRP-300 Printing Results
- **Images**: Print correctly via `bitImageColumn` implementation with `ESC 3 0` (zero line spacing)
- **Paper cut**: Best results with `GS V 65 3` (`1D 56 41 03`) — 3-line feed then full cut. Clean cuts achieved.
- **Top margin (~1.7cm)**: This is the physical cutter-to-print-head distance. **Cannot be reduced via software** — the PRP-300 does not support reverse feed (`ESC j`, `ESC e`, `ESC K` all ignored) and has no DIP switch or memory switch for top margin reduction. Only workaround is padding the image bottom to match.
- **USB disconnect issue**: Printer sometimes disconnects after large prints; reconnects automatically. Cable sensitivity observed.

### 3. Citizen CT-S310II Print Optimization (main branch)
- **White border trimming**: Added `ImageOps.invert` + `getbbox()` crop before scaling to remove empty margins
- **Trim before scale**: Moved trim step before resize so forced dimensions fill the full print area
- **Forced resize**: Changed from `thumbnail()` (aspect-preserving) to `resize()` (exact dimensions, allows distortion) so images fill the entire print area
- **Calibrated print dimensions**:
